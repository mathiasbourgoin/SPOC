Kernel Type

type 'a kernel = {
  1. name : string;
    (*

    Kernel name (used for compilation cache key)

    *)
  2. ir : Sarek_ir_types.kernel Stdlib.Lazy.t;
    (*

    Lazy IR generation - only forced for JIT backends

    *)
  3. native_fn : (block:Spoc_framework.Framework_sig.dims -> grid:Spoc_framework.Framework_sig.dims -> Spoc_framework.Framework_sig.exec_arg array -> unit) option;
    (*

    Pre-compiled OCaml function for Direct backends

    *)
  4. param_types : Sarek_ir_types.elttype list;
    (*

    Parameter types for argument marshalling

    *)
  5. extensions : Kirc_types.extension array;
    (*

    Extensions required (ExFloat32, ExFloat64)

    *)
}

Phase 4 kernel record with lazy IR generation

Execution

val run : device:Spoc_core.Device.t -> block:Spoc_framework.Framework_sig.dims -> grid:Spoc_framework.Framework_sig.dims -> ?shared_mem:int -> 'a kernel -> Spoc_framework.Framework_sig.exec_arg array -> unit

Execute a kernel on a device with exec_arg array args. Note: Only works for Native backend. For JIT backends (CUDA/OpenCL), use run_with_args which provides properly typed arguments.

val run_with_args : device:Spoc_core.Device.t -> block:Spoc_framework.Framework_sig.dims -> grid:Spoc_framework.Framework_sig.dims -> ?shared_mem:int -> 'a kernel -> Execute.vector_arg list -> unit

Execute a kernel with explicit typed arguments. Works for all backends (Native, CUDA, OpenCL). Uses plugin dispatch via Execute.run.
